Space Invaders Refactor

Solo | Software Developer C++ | 1 week

Given a Space Invaders Project, I refactored their code to include idiomatic modern C++, clean code and good api design.

Core issue

Upon reviewing the codebase, this project's core issue is the constant violation of Single Responsibility Principle. Here are a few indicators of how the codebase violates the principle:

Physical structure

In this project, there are only 3 .cpp files

Logical structure

The game.cpp file holds all the objects involved with the game (Projectile, Wall, etc...)

As a result, game.cpp was 939 lines long

Three-step initialization

A lot of objects would be created in a local variable, with a couple of lines changing special parameters, and in some cases, calling a launch function.

The object itself isnâ€™t handling itself, but another object.

Immediate solutions

One file per object

Give every object it's own file

game.cpp became 367 lines, handling less responsibility

More Functions

Package all meaningful operations into functions so that the core functions (Game::Update()) becomes more readable

(C++ Core Guidelines F.1)

Group functions

All rendering functions can be grouped into a new class that handles solely rendering, making game.cpp 223 lines

Better Constructors

Each object minimizes the number of dynamic variables needed for object creation, and use constructors so that creation only needs one line.

More improvements

RAII

Ensure critical resources such as texture is handled properly with constructor, destructor, and exception specifying.

This ensures that the object is always created in a valid state

Pipelined logic

Reduce complexity by keeping only essential data, and relying more on simple calculations

e.g. projectile doesn't need to keep updating a line start/end, just a position that renderer will draw out.

Modern C++

- Adhering to the rule of 0
- for-range loops
- static_cast over c-style cast if necessary
- using std::algorithms (remove_if(), for_each(), etc...)
- auto, [[nodiscard]], constexpr, noexcepts

After this refactor, game.cpp turned from 939 to 18 lines.

This shows how Single Responsibility Principle is a lot less violated.

Readability

Because all meaningful operations and values have been stored, the code reads like English.

Modern C++

Using C++ Core Guidelines, textbooks and C++ Convention Talks, made motivated decisions to improve the code

How to further improve this?

State Machine

Further adhere to Single Responsibility Principle by implementing a state machine handling different states

Bug in OwnTexture

The special member operations are not handling all the members of the class, which means a copied texture will take an undefined _path.

Greater optimisation

The code could be further optimised. Examples include using auto-lambda templates, string_view, less unecessary copy with emplace_back
