Technical Challenge:
Feathering Circles

A huge mechanic of Luminth was the uncovering of the maze through lights. However, Unity doesn't have a built-in function to create transparent circles, needing technical expertise.

Original submission

Within 72 hours, the best solution was using the stencil buffer, to reject pixels on the cloud texture based on a circle image attached with the light

Polished Version

Using HLSL, we not only use alpha blending on the cloud texture, we also draw the player's silhouette accordingly.

How was feathering achieved?

1. Connect the cloud texture material with the .shader file
2. Using C#, Convert worldspace coordinates to the cloud texture's UV coordinates
3. Share the necessary data to render all lights (points and radius) and the player (texture and position).
4. First, draw the pixels in which the player would be on the cloud texture, in a grey color.
5. Using Signed Distance Fields, define a HLSL drawCircle function that uses distance to determine which pixels are in the circle
6. in drawCircle, feather a portion of the circle inside using smoothstep
7. Render the cloud texture, using the value from drawCircle as the alpha.

New: Focus Mechanic

From the original submission's feedback, we have introduced a new mechanic that helps players find the direction of the exit.

Pseudocode

1. Connect a new image with the .shader file
2. Share the necessary position datas of the exit, player, and cursor.
3. Using Desmos as a visualizer, construct a wave on a circle equation that takes angle and amplitude variables. The wave = direction to exit from player * cursor direction from unit circle
4. Using Signed Distance Fields, define a drawCircle function that uses distance to determine which pixels are in the wave/circle
5. Using a delayed cursor position, create chromatic aberration by passing in different values in the red green blue channels.
6. Draw a black inner circle to show the character.
7. Trigger this image using a button press in Unity.

Improved Tutorial

Original submission

Due to the interesting world Luminth had to offer, the original tutorial focused more on narrative to introduce the mechanics. However, we found that it gave players unclear instructions and expectations

Polished Version

In the polished version, Luminth's tutorial dialogues went from 27 lines to 5 lines. This gave more focus on action and visual instructions than words, making it clearer what players should expect.

Key Components

Level Design

Using my knowledge of level design, I aimed to make a tutorial level that felt intuitive yet informative.

Game Events

Using Game Events, I only needed to listen to triggers and respond to them in script, reducing dependencies.

Designer Friendly

Because the script was responsive to Game Events and the method call was the dialogue line, it was easy to let others add to it.
